# 德州扑克机器人AI功能文档

## 概述

德州扑克游戏已成功集成AI机器人功能，支持1个真人玩家与多个AI机器人对战。机器人通过Coze智能体API进行决策，具备智能的扑克策略。

## 功能特性

### 1. 机器人配置
- ✅ 每个机器人可以独立配置
- ✅ 支持不同的AI模型
- ✅ 每个机器人使用自己的API Token
- ✅ 自动决策，无需人工干预

### 2. AI模型支持
支持以下Coze AI模型：
- **豆包 Seed 1.8** (默认) - 平衡性能和智能
- **豆包 Seed 2.0 Lite** - 高性价比
- **豆包 Seed 2.0 Mini** - 快速响应
- **DeepSeek V3.2** - 高级推理
- **Kimi K2.5** - 长上下文处理

### 3. 智能决策
机器人会根据以下因素做出决策：
- 手牌强度
- 公共牌组合
- 底池大小
- 筹码数量
- 对手行为
- 游戏阶段

## 使用步骤

### 1. 获取Coze API Token

1. 访问 [Coze 官网](https://www.coze.cn/)
2. 登录并进入个人中心
3. 找到"个人访问令牌 (PAT)"或"API Token"
4. 创建一个新的Token或使用已有的Token
5. 复制Token（格式：`pat_xxxxxx`）

### 2. 配置机器人

1. 进入游戏设置页面
2. 选择玩家数量（2-9人）
3. 配置每个机器人玩家：
   - 输入机器人名称
   - 输入Coze API Token（必填）
   - 选择AI模型
4. 确保至少有一个真人玩家

### 3. 开始游戏

1. 点击"开始游戏"按钮
2. 系统会自动初始化游戏
3. 真人玩家和机器人轮流行动
4. 机器人会自动做出决策

### 4. 观察机器人行为

机器人决策会显示在游戏控制台中：
```
[23:45:10] 轮到: 机器人 Alice
[23:45:11] 机器人正在思考...
[23:45:13] 机器人决策: call 20
[23:45:13] [机器人 Alice] call
```

## 机器人决策类型

机器人可以执行以下操作：

### 1. 弃牌 (Fold)
**触发条件：**
- 手牌很弱
- 底池已经很大
- 跟注成本过高

**示例：**
```
机器人决策: fold
```

### 2. 过牌 (Check)
**触发条件：**
- 当前下注额等于自己的下注额
- 手牌中等
- 想要免费看牌

**示例：**
```
机器人决策: check
```

### 3. 跟注 (Call)
**触发条件：**
- 需要跟注的金额可接受
- 手牌有一定价值
- 底池赔率合适

**示例：**
```
机器人决策: call 20
```

### 4. 加注 (Raise)
**触发条件：**
- 手牌很强
- 想要增加底池
- 诈唬或施压对手

**示例：**
```
机器人决策: raise 40
```

### 5. 全押 (All In)
**触发条件：**
- 筹码很少
- 手牌非常好
- 想要最大化收益

**示例：**
```
机器人决策: allin
```

## 技术实现

### 1. 机器人AI服务

位于 `server/src/services/bot-ai.service.ts`，负责：
- 调用Coze API获取决策
- 解析智能体返回
- 处理错误和超时

### 2. API接口

**机器人决策API：**
```
POST /api/game/bot-action
```

**请求参数：**
```json
{
  "gameId": "game-xxx"
}
```

**返回参数：**
```json
{
  "success": true,
  "data": {
    // 更新后的游戏状态
  },
  "decision": {
    "action": "call",
    "amount": 20,
    "reason": "手牌不错，底池赔率合适"
  }
}
```

### 3. 前端集成

- 自动检测当前玩家是否是机器人
- 延迟1-2秒模拟思考时间
- 调用机器人决策API
- 更新游戏状态

### 4. 决策提示词

机器人使用以下提示词进行决策：

```
你是一个专业的德州扑克玩家。请根据当前游戏状态，做出最佳的决策。

你的回复必须严格遵循以下JSON格式：
{
  "action": "fold|check|call|raise|allin",
  "amount": 数字（仅在raise时需要）,
  "reason": "简短的原因说明"
}

决策规则：
1. 如果手牌很差且底池已经很大，选择fold弃牌
2. 如果当前下注额等于你的下注额，可以选择check过牌
3. 如果需要跟注且手牌还不错，选择call跟注
4. 如果手牌很好且想加注，选择raise加注
5. 如果筹码很少且手牌不错，可以选择allin全押
```

## 注意事项

### 1. API Token安全
- 不要在代码中硬编码API Token
- 定期更换API Token
- 不要在公共场合暴露Token

### 2. 机器人数量
- 最多支持8个机器人（加上真人共9人）
- 每个机器人都需要独立的API Token

### 3. 思考时间
- 机器人会自动延迟1-2秒模拟思考
- 不支持自定义思考时间

### 4. 错误处理
- 如果机器人决策失败，会使用默认保守策略
- 错误会记录在游戏控制台中

### 5. 数据库记录
- 机器人操作会记录到数据库
- 可以查询历史操作日志

## 性能优化

### 1. 客户端缓存
机器人AI服务使用客户端缓存，避免重复创建连接。

### 2. 异步处理
机器人决策是异步的，不会阻塞游戏进程。

### 3. 超时设置
机器人决策超时时间为30秒，超时后使用默认策略。

## 常见问题

### Q1: 机器人决策很慢怎么办？
A: 可以选择更快的AI模型，如"豆包 Seed 2.0 Mini"。

### Q2: 机器人总是弃牌？
A: 检查API Token是否正确，确保有足够的调用额度。

### Q3: 可以让机器人更激进吗？
A: 可以通过调整AI模型的temperature参数（目前固定为0.7）。

### Q4: 机器人能看到其他玩家的手牌吗？
A: 不能，机器人只能看到自己的手牌和公共牌。

### Q5: 可以使用自己的智能体吗？
A: 可以，在BotAIService中修改system prompt来定制机器人的策略。
